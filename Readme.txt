Driving Log Analysis 포트폴리오 (Python)

이 저장소는 차량 주행 로그 데이터를 기반으로
급가속과 급제동 이벤트를 탐지하고,
이를 통해 운전자의 주행 성향과 위험도를 분석하는 데이터 분석 프로젝트 모음이다.

단순히 그래프를 그려보는 수준이 아니라,
자동차 시험과 QA, 주행 데이터 검증 관점에서
실제 현업에서 사용할 수 있는 로직을 구현하는 것을 목표로 했다.

프로젝트 구성 개요

전체 흐름은 주행 로그 데이터의 기초 분석에서 시작한다.
이후 이벤트 탐지, 주행 성향 분류, 점수화 모델로 단계적으로 확장된다.

각 프로젝트는 이전 단계의 결과를 기반으로 이어지며,
분석 복잡도는 높아지지만 해석 가능성은 유지하도록 설계했다.

프로젝트 목록
프로젝트 01. 주행 로그 기본 분석

원시 주행 로그 데이터의 구조를 파악하는 단계다.
시간 정보를 기준으로 속도와 가속도를 계산하고,
기본적인 주행 패턴을 시각화해 데이터 이상 여부를 확인한다.

문서: project-01-driving-log-analysis.md

프로젝트 02. 급가속 · 급제동 이벤트 탐지

가속도 임계값을 기준으로 급가속과 급제동 이벤트를 정의한다.
주행 로그 전체에서 해당 이벤트를 자동으로 탐지하고,
이벤트가 발생한 구간을 그래프로 확인할 수 있도록 구성했다.

문서: project-02-hard-event-detection.md

프로젝트 03. 운전자 주행 성향 분류

급가감속 이벤트의 발생 빈도를 기준으로 운전자의 주행 성향을 분석한다.
단순 이벤트 개수뿐 아니라 시간당, 거리당 빈도 지표를 함께 사용했다.

주행 시간이 지나치게 짧거나 주행 거리가 부족한 경우
빈도 값이 과대평가되는 문제를 방지하기 위한 로직도 포함했다.

문서: project-03-driver-behavior-classification.md

프로젝트 04. 공격적 주행 점수 모델

여러 주행 지표를 하나의 점수로 통합하는 모델이다.
급가감속 빈도, 급제동 비율, 속도 변동성을 함께 반영해
0에서 100 사이의 공격적 주행 점수를 산출한다.

운전자 간 비교나 주행 세션 간 비교를 염두에 두고 설계했다.

문서: project-04-aggressiveness-score.md

프로젝트 05. ADAS 개입 필요도 점수 모델

앞선 프로젝트들의 결과를 종합해 ADAS 개입 필요도를 수치화한다.
경고 수준과 제어 개입 수준을 구분하기 위한 기준 로직을 설계했다.

통계 모델보다는
ADAS 시험과 검증 관점의 판단 로직에 초점을 맞췄다.

문서: project-05-adas-intervention-score.md

이 프로젝트는 순수한 데이터 분석이 아니라
ADAS 검증 관점에서 설계되었다.

기술 스택

분석은 Python을 중심으로 진행했다.
데이터 처리에는 Pandas와 NumPy를 사용했고,
시각화는 Matplotlib을 활용했다.
분석 과정은 주로 Jupyter Notebook 환경에서 수행했다.

활용 목적

차량 주행 시험 데이터를 분석하는 용도로 활용할 수 있다.
자동차 QA 및 Validation 로직 검증,
운전자 주행 성향 평가,
ADAS 및 안전 시스템 로직 설계 시 참고 지표로 사용할 수 있다.

프로젝트 특징

실제 차량 주행 로그 형식을 기준으로 분석을 진행했다.
프로젝트가 단계적으로 확장될 수 있도록 구조를 설계했다.
현업 시험과 QA, ADAS 관점에서 문제를 정의하고 해결했다.
데이터 부족으로 인한 과대평가를 방지하는 안정성 로직을 포함했다.
